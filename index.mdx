Podman is a fully featured container engine that runs without a background daemon — giving you improved security, lower resource usage at idle, and a Docker-compatible CLI you can adopt immediately. Run containers as a non-root user, manage groups of containers as pods, and integrate with systemd natively through Quadlet.

## Why Podman?

Podman is designed as a secure, flexible alternative to Docker with full OCI compliance and no mandatory daemon process.

## Get up and running

<Steps>
  <Step title="Install Podman">
    Install Podman for your platform. On Fedora/RHEL: `sudo dnf install podman`. On Ubuntu/Debian: `sudo apt install podman`. On macOS: `brew install podman`.
  </Step>
  <Step title="Run your first container">
    Pull and run a container in one command:
    ```bash
    podman run -it docker.io/library/alpine sh
    ```
  </Step>
  <Step title="List running containers">
    See what's running with:
    ```bash
    podman ps
    ```
  </Step>
  <Step title="Explore pods and volumes">
    Group containers into pods or persist data with named volumes — see the [Guides](/guides/containers) section for full walkthroughs.
  </Step>
</Steps>

## Key features

<AccordionGroup>
  <Accordion title="Daemonless architecture">
    Podman does not require a long-running background daemon. Each `podman` command runs as a direct child process, reducing your attack surface and idle resource usage. There is no single point of failure.
  </Accordion>
  <Accordion title="Docker-compatible CLI">
    Podman's command-line interface is designed to be a drop-in replacement for Docker. Most users can `alias docker=podman` and continue working without changes. Podman also supports Docker Compose workflows via `podman compose`.
  </Accordion>
  <Accordion title="OCI and Docker image formats">
    Podman supports both OCI image format and Docker image format. Pull from any OCI-compliant registry, including Docker Hub, Quay.io, GitHub Container Registry, and private registries.
  </Accordion>
  <Accordion title="REST API">
    Podman exposes a REST API compatible with the Docker API as well as a Podman-native API with extended functionality. Use the Go bindings (`pkg/bindings`) to integrate Podman operations into your Go applications.
  </Accordion>
  <Accordion title="Cross-platform support">
    Run Podman natively on Linux. On macOS and Windows, Podman manages a lightweight virtual machine (`podman machine`) that provides a Linux kernel for running containers, with seamless CLI integration.
  </Accordion>
</AccordionGroup>
